1. Overview

The news pipeline ingests RSS feeds from 7 gaming publications every 10 minutes, normalizes and deduplicates articles, clusters related stories, links articles to game entities via RAWG, and computes per-game trending scores every 30 minutes. The mobile app reads directly from Supabase.

Sources (7)

IGN, GameSpot, Eurogamer, PC Gamer, Gematsu, PlayStation Blog, Xbox Wire — all RSS 2.0 / Atom feeds

Schedules

ingest-news every 10 min · calculate-news-trends every 30 min · pg_cron + net.http_post

Game matching

Local alias cache first → local name lookup → RAWG API fallback with confidence scoring

Security

X-Cron-Secret header on all cron calls · all credentials in Supabase Vault · fail-closed auth guard

3. Ingestion Function

File: supabase/functions/ingest-news/index.ts

Atomic source claiming

Prevents two concurrent workers from fetching the same source. Uses UPDATE ... RETURNING * to advance next_fetch_at to a 30-minute lease before fetching: 

-- Equivalent SQL (done via Supabase client)
UPDATE news_sources
SET    next_fetch_at = now() + interval '30 minutes'   -- lease
WHERE  is_enabled = true
  AND  next_fetch_at <= now()
RETURNING *;

After a successful fetch, updateSourceState() overwrites next_fetch_at with the real next schedule plus up to 10 min of random jitter.

Per-source error handling

HTTP statusAction
304 Not ModifiedSkip parse, update ETag/Last-Modified, reset failures
429 Rate LimitedBack off 6 hours, increment consecutive_failures
5xx / network error≥3 failures → disable source + 12h pause; otherwise 1h pause

Retry-safe game matching

Matching is decoupled from insertion. After each source fetch, matchUnmatchedArticles() queries all articles from the last 72 hours that have no news_article_games row and runs the RAWG matcher on them. This retries articles that were stored during a RAWG outage or missing API key. 

-- Finds unmatched articles regardless of current run
SELECT id, title, excerpt
FROM   news_articles
WHERE  source_id = $1
  AND  fetched_at >= now() - interval '72 hours'
  AND  id NOT IN (SELECT article_id FROM news_article_games);

4. RSS Parsing & Validation

File: supabase/functions/ingest-news/parser.ts

Uses npm:fast-xml-parser@4.3.6 in Deno. Handles both RSS 2.0 and Atom.

URL trust check

Every article URL from a feed is validated before storage. Rejects http:// and URLs whose root domain doesn't match the source's homepage_url — prevents malicious feed entries from storing arbitrary URLs. 

// isUrlTrusted(): HTTPS required + root domain must match source homepage
rootDomain('https://www.ign.com/articles/...')  // → "ign.com"
rootDomain('https://ign.com')                  // → "ign.com"  ✓ trusted
rootDomain('https://evil.com')                 // → "evil.com" ✗ rejected

Content hash

Each article gets a SHA-256 hash of its normalized title for deduplication. upsertArticles() uses ON CONFLICT (url) DO NOTHING — existing articles are never overwritten.

5. Story Clustering

File: supabase/functions/ingest-news/clusterer.ts

Groups articles about the same story across sources. Runs at the end of every ingest cycle over a 72-hour window.

Algorithm

  1. Fetch unclustered articles from the last 72 hours.
  2. Load article → game matches for those articles.
  3. Load existing recent cluster → game matches with an embedded PostgREST filter.
  4. Compare normalized titles with Levenshtein similarity.
  5. If both article and cluster have matched games, require at least one overlapping game_id.
  6. If similarity ≥ 0.72 and the game constraint passes, assign the article to that cluster.
  7. Create a new cluster for unmatched articles.

Article freshness rule

Clustering and ranking use the shared freshness helper effectiveArticleTime(article): 

published_at ?? fetched_at ?? created_at

Homepage story score

Recent clusters are scored at the end of each ingest pass. Reads are batched by cluster set (clusters, articles, article-game links, and game trends), then each cluster is updated with only story_score and updated_at to avoid overwriting concurrent cluster metadata. 

story_score =
  recency_score
  + best_source_weight × 10
  + min(max_game_trending_score / 10, 25)
  + unique_source_count × 5
  - same_source_duplicate_count × 5
Note

The cluster join still uses title similarity as the main heuristic. Game overlap is a guardrail when both sides have resolved games; articles without a game match can still cluster by title alone.

6. Game Matching (RAWG)

File: supabase/functions/ingest-news/gameMatcher.ts

Three-tier cascade, fastest first:

TierSourceCondition
1Local alias table (news_game_aliases)Exact normalized title match
2Local game name (news_games.name)Exact normalized match
3RAWG APINo local hit; RAWG_API_KEY must be set

Candidate extraction

Before searching, extractCandidates() pulls game name candidates from title + excerpt:

  • Text before keywords: review, preview, trailer, hands-on, announced…
  • Capitalized multi-word phrases (≥2 words)

RAWG confidence scoring

Score thresholdAction
≥ 0.85high Confident match — upsert game + save alias
0.65 – 0.84medium Queue in news_game_match_candidates for manual review
< 0.65Discard — too ambiguous

Manual review queue

Low-confidence RAWG candidates are stored in news_game_match_candidates with the best RAWG hit, confidence score, raw RAWG result payload, and status = 'pending'. The table is intentionally service-role only: RLS is enabled with no public policies until an admin review UI exists.

RAWG key missing

If RAWG_API_KEY is unset, searchRawg() returns empty silently. Articles remain unmatched and will be retried on the next ingest cycle via matchUnmatchedArticles().

7. Trending Scores

File: supabase/functions/calculate-news-trends/index.ts

Runs every 30 minutes. Writes one row per game to news_game_trends.

Freshness windows

Trend windows are based on article freshness, not the audit timestamp on the join table. The function loads a DB-bounded 14-day candidate set from news_article_games.created_at, then refines all 24h / 72h / 7d / previous-7d windows in memory using: 

coalesce(news_articles.published_at, news_articles.fetched_at, news_articles.created_at)

This keeps the Edge Function bounded while preserving the correct product model: published_at for content freshness, fetched_at as fallback, and news_article_games.created_at for audit/bounding only.

Formula

trending_score =
  mentions_24h      × 4
  + mentions_72h    × 2
  + unique_sources_72h × 8
  + official_mentions_72h × 10   -- PlayStation Blog / Xbox Wire only
  + spike_bonus                  -- +15 if mentions_24h ≥ 3 AND 7d prior avg ≤ 1

Official sources

OFFICIAL_SOURCE_IDS = { 'playstation-blog', 'xbox-wire' } — first-party announcements carry 10× weight because they signal real news (release dates, exclusives) vs. editorial coverage.

8. Database Schema

News read tables have RLS enabled with public read policies. Writes only via service role (Edge Functions). news_game_match_candidates is service-role only and has no public policies.

TablePurposeKey columns
news_sources RSS feed registry id, feed_url, homepage_url, next_fetch_at, consecutive_failures, is_enabled
news_articles Normalized article store id, url (unique), title, excerpt, published_at, fetched_at, source_id, content_hash
news_story_clusters Cross-source story groups id, representative_title, normalized_title, story_score, article_count, unique_source_count, latest_published_at
news_games Resolved game entities id, rawg_id (unique), name, slug, image_url, platforms, genres
news_game_aliases Alternate names / RAWG hits cache game_id, alias, source (rawg_match)
news_article_games Article ↔ game join article_id, game_id, confidence, match_method, created_at
news_game_match_candidates Manual review queue for low-confidence RAWG matches article_id, candidate, best_rawg_id, confidence, rawg_results, status
news_game_trends Per-game trending scores game_id, trending_score, mentions_24h, mentions_72h, official_mentions_72h, calculated_at

How cron auth works

pg_cron fires a net.http_post that reads all three secrets from Vault at execution time — no credentials are baked into the scheduled job string. The Edge Function checks X-Cron-Secret against CRON_SECRET env var and fails closed (returns 401) if missing or wrong. 

-- Cron command (simplified)
SELECT net.http_post(
  url     := (SELECT decrypted_secret FROM vault.decrypted_secrets WHERE name = 'supabase_functions_url') || '/ingest-news',
  headers := jsonb_build_object(
    'Authorization', 'Bearer ' || (SELECT decrypted_secret FROM vault.decrypted_secrets WHERE name = 'supabase_anon_key'),
    'X-Cron-Secret', (SELECT decrypted_secret FROM vault.decrypted_secrets WHERE name = 'cron_secret')
  ),
  body := '{}'::jsonb
);
New environment

Applying migrations to a new project (local, staging, restored DB) requires creating all three Vault secrets first. Migration 005 will fail loudly with a named exception rather than silently scheduling broken jobs.

10. Mobile Hooks & Pages

The mobile app reads directly from Supabase through React Query hooks. There are no separate /api/news/* routes in the current implementation.

Hook / pagePurpose
useStoryClusters / GamingNewsHomepage Top Stories, ordered by story_score then latest publish time
useLatestNews / /news/latestIndividual articles, newest first, paginated
useTrendingGames / /news/trending-gamesTop games by news_game_trends.trending_score
useNewsForGame / /news/game/[slug]Articles linked to a resolved news game entity

Latest news query shape:

// Fetches articles, most recent first
const { data, error } = await supabase
  .from('news_articles')
  .select('id, title, url, excerpt, author, published_at, source_id, news_sources(name)')
  .order('published_at', { ascending: false, nullsFirst: false })  // nulls go last
  .order('fetched_at', { ascending: false })                       // fallback for undated
  .limit(30)

React Query config for news-facing hooks: staleTime: 10min, gcTime: 30min.

11. Monitoring Queries

Run these in the Supabase SQL editor to check pipeline health.

Cron job history

SELECT
  j.jobname,
  d.status,
  d.return_message,
  d.start_time,
  d.end_time - d.start_time AS duration
FROM cron.job_run_details d
JOIN cron.job j ON j.jobid = d.jobid
WHERE j.jobname IN ('ingest-news', 'calculate-news-trends')
ORDER BY d.start_time DESC
LIMIT 10;

Article ingestion rate

SELECT
  s.name,
  COUNT(a.id)             AS total_articles,
  MAX(a.fetched_at)       AS last_fetched,
  s.consecutive_failures,
  s.is_enabled
FROM news_sources s
LEFT JOIN news_articles a ON a.source_id = s.id
GROUP BY s.id
ORDER BY last_fetched DESC NULLS LAST;

Unmatched articles (RAWG gaps)

SELECT
  a.title,
  a.source_id,
  a.fetched_at
FROM news_articles a
WHERE a.fetched_at >= now() - interval '72 hours'
  AND NOT EXISTS (
    SELECT 1 FROM news_article_games g WHERE g.article_id = a.id
  )
ORDER BY a.fetched_at DESC;

Top trending games right now

SELECT
  ng.name,
  t.trending_score,
  t.mentions_24h,
  t.mentions_72h,
  t.official_mentions_72h,
  t.calculated_at
FROM news_game_trends t
JOIN news_games ng ON ng.id = t.game_id
ORDER BY t.trending_score DESC
LIMIT 20;

Top ranked story clusters

SELECT
  representative_title,
  story_score,
  article_count,
  unique_source_count,
  latest_published_at
FROM news_story_clusters
ORDER BY story_score DESC, latest_published_at DESC NULLS LAST
LIMIT 20;

Pending game match review queue

SELECT
  c.candidate,
  c.best_rawg_name,
  c.confidence,
  a.title,
  c.created_at
FROM news_game_match_candidates c
JOIN news_articles a ON a.id = c.article_id
WHERE c.status = 'pending'
ORDER BY c.created_at DESC
LIMIT 50;

Vault secrets present

SELECT name, created_at
FROM vault.decrypted_secrets
WHERE name IN ('cron_secret', 'supabase_functions_url', 'supabase_anon_key')
ORDER BY name;
